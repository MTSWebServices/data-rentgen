Added parent relation between jobs. Jobs can now reference a parent job via ``parent_job_id`` field. The ``GET /v1/jobs`` endpoint accepts new ``parent_job_id`` query parameter to filter jobs by parent, and job responses include ``parent_job_id`` (null when job has no parent).


  .. dropdown:: Jobs response example

    Before:

    .. code-block:: json

      {
        "meta": { ... },
        "items": [
          {
            "id": "42",
            "name": "my-spark-task",
            "type": "SPARK_APPLICATION",
            "location": { "id": "1", "type": "yarn", "name": "my-cluster", "addresses": [], "external_id": null }
          }
        ]
      }

    After:

    .. code-block:: json
      {
        "meta": { ... },
        "items": [
          {
            "id": "42",
            "parent_job_id": "10",
            "name": "my-spark-task",
            "type": "SPARK_APPLICATION",
            "location": { "id": "1", "type": "yarn", "name": "my-cluster", "addresses": [], "external_id": null }
          }
        ]
      }


Job without parent returns ``parent_job_id`` as ``null``.
