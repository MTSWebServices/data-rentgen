Added ``ancestors`` field to lineage ``relations`` response. It contains run→run ancestry chains (e.g. Airflow DAG→Task→Spark app). The existing ``parents`` field (job→run and run→operation hierarchy) remains unchanged.

  .. dropdown:: Lineage relations response example

    Before:

    .. code-block:: json

        {
            "relations": {
                "parents": [
                    {"from": {"kind": "JOB", "id": "1"}, "to": {"kind": "RUN", "id": "run-uuid"}},
                    {"from": {"kind": "RUN", "id": "run-uuid"}, "to": {"kind": "OPERATION", "id": "op-uuid"}}
                ],
                "symlinks": [],
                "inputs": [...],
                "outputs": [...]
            },
            "nodes": {...}
        }

    After:

    .. code-block:: json

        {
            "relations": {
                "parents": [
                    {"from": {"kind": "JOB", "id": "1"}, "to": {"kind": "RUN", "id": "run-uuid"}},
                    {"from": {"kind": "RUN", "id": "run-uuid"}, "to": {"kind": "OPERATION", "id": "op-uuid"}}
                    {"from": {"kind": "RUN", "id": "parent-run-uuid"}, "to": {"kind": "RUN", "id": "run-uuid"}}
                ],
                "symlinks": [],
                "inputs": [...],
                "outputs": [...]
            },
            "nodes": {...}
        }
